## 問題の根本原因

JSON データベースの `articleNum` フィールドが、法律の条文番号のサフィックス部分を正しく記録していない。

### 日本の法律における条文番号の構造
- 基本形: 第X条 (例: 第二十七条)
- サフィックス付き: 第X条のY (例: 第二十七条の二、第三十一条の二)

### 現在の JSON 構造の問題
```javascript
// 誤った例 (現状)
{
  "articleNum": "二十七",  // サフィックスなし
  "title": "広告宣伝の禁止"  // 実際は第二十七条の二
}

// 正しい例 (あるべき姿)
{
  "articleNum": "二十七の二",  // サフィックス含む
  "title": "広告宣伝の禁止"
}
```

## 推奨される対応

### 1. 第4-6章の構造修正 (優先度: 高)
- 各記事の実際の条文番号をPDFから確認
- articleNum フィールドにサフィックスを追加
- 重複を解消し、一意の記事番号を設定

### 2. 施行規則の全章検証 (優先度: 高)
- 第1章以降の全記事番号を PDFと照合
- 異常値 (例: "二百三十四") を修正

### 3. データ整合性チェックの実装 (優先度: 中)
- 同一章内での記事番号重複を検出するバリデーション
- 記事番号の連続性チェック
