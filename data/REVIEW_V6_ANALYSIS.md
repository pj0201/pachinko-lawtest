## 🔴 **発見された重大な問題**

### 1. **【致命的】ロジック矛盾 - 答え方向が逆転**

#### 問題例1:
```
Q: 【誤ったルール】営業開始前に許可を得なければならない。
答: ✕ FALSE
テスト目的: 法定ルールの正確な知識
```

**評価**: ❌ **INVALID - 大問題**
- 【誤ったルール】と明記されているが、この文は **実は正しいルール** です
- 答えを FALSE にすると「誤ったルール = 正しい」という矛盾
- テスト者に混乱を招く

#### 問題例2:
```
Q: 【法定ルール】営業開始前に許可を得なければならない。
答: ✕ FALSE
```

**評価**: ❌ **INVALID**
- 【法定ルール】と明記されているのに答えが FALSE
- これは完全に逆です
- テスト無効

## 📋 改善提案

### 即座の修正が必要

1. **ロジック修正** (最優先):
   - 問題文の【誤ったルール】と答え(FALSE)のロジックを統一
   - または問題文の【】プリフィックスの使い分けを明確化
   - テスト実施前に全問を自動検証

2. **フォーマット分布の均等化**:
   - 各フォーマットを約117-120問ずつに調整
   - ランダムフォーマット選択の代わりに、計画的にラウンドロビン割り当て

3. **生成問題数の拡大**:
   - 補完ロジックを強化
   - ユニーク性チェックの基準を調整
   - 反復試行上限を増加
