## ⚠️ 重要な発見

### 修正戦略の制約

1. **テーブル構造の喪失**
   - 多くの混入はテーブルセル分割の失敗に由来
   - 英字を単純に削除するだけでは不十分
   - 原データ復元には原PDF確認が必須

2. **複合的OCR失敗**
   - 英字混入 + 日本語誤字の組み合わせ
   - 例: "pmーleaooumeemののmoeemlooae"
   - 単純な削除では意味不明な状態に

3. **法律文書の重要性**
   - 正確性が重要な法律関連テーブル
   - 不正確な修正は問題を招く可能性

## 📊 修正可能性の分類

### ✅ 確実に修正可能（単純な英字除去）

```
パターン: 単一英字が日本語に埋め込まれたもの
例: 'leaooumeemの' → 'の'（ただし不完全）
件数: 約20%（~28件）
リスク: 低～中

実例:
- ページ43: 'leaooumeemの'
- ページ44: 'uょ', 'やplpnsmosrlwe'
```

### ⏳ 文脈依存（要人間判定）

```
パターン: 複数英字混入＋文脈破損
例: 'Floesロles uょらやplpnsmosrlwe選の'
件数: 約50%（~70件）
リスク: 中～高

対応方法:
1. 周辺テキストの文脈確認
2. 法律用語の規定内容確認
3. 原意の推測
```

### ❌ 修正不可（原PDF必須）

```
パターン: テーブル全体の破損＋大量混入
例: 'ょemmlmeiePロle oeFmI]*や'
件数: 約30%（~42件）
リスク: 高

対応方法:
- 原PDFのテーブル画像確認必須
- OCR不信頼セクション マーク
```
